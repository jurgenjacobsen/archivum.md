# PLAN.md Execution Method
When prompted to execute the PLAN provided execute the following steps:
1. **Understand the PLAN**: Carefully read and comprehend the PLAN provided. Ensure you understand each step and its purpose. If there are any unclear points, ask for clarification before proceeding.
2. **Understand the environment**: Familiarize yourself with the codebase in which you will be executing the PLAN. This includes understanding the structure of the code, the libraries being used, and any relevant documentation.
3. **Identify the tasks**: Break down the PLAN into individual tasks or steps. If no information provided in the prompt follow the plan from the unchecked items/tasks in the PLAN.md file. If provided in the prompt only execute the tasks mentioned in the prompt.
4. **Task Management**: If a certain task requires multiple steps, break it down further into sub-tasks. This will help in managing the execution process more effectively. Add the sub-tasks/steps to the PLAN.md file under the corresponding task.
5. **Seamless Addition of Tasks**: The features or code changes to the codebase should try to follow the existing code style and structure as much as possible. If there are any specific guidelines or best practices mentioned in the codebase documentation, make sure to adhere to them while adding new features or making changes. If you think that a different approach would be more effective or efficient, feel free to suggest it and discuss it with the team before implementing it. Any addition to the UI should be seamless and should not disrupt the user experience. It should blend in with the existing design and functionality of the application. Always consider the end-users while making changes to the UI and ensure that it enhances their experience rather than complicating it.
6. **Execute the tasks**: Start executing the tasks one by one. Ensure that you follow the instructions provided in the PLAN and adhere to any guidelines or constraints mentioned.
7. **Testing**: After executing each task, test the changes to ensure that they work as expected and do not introduce any new issues or bugs.
8. **Document the process**: As you execute each task, document your process and any challenges you encounter. This will help in tracking your progress and can be useful for future reference.
9. **Seek feedback**: After completing the tasks, seek feedback from your team to ensure that the changes meet the requirements and expectations.
10. **Iterate and improve**: Based on the feedback received, make any necessary adjustments or improvements to the changes you have made. This may involve going back to previous steps and re-executing certain tasks to ensure that the final outcome is of high quality and meets the desired standards.
11. **Final Review**: Once all tasks are completed and feedback has been incorporated, conduct a final review of the changes to ensure that everything is in order and ready for deployment or integration into the main codebase.
12. **Log Changes**: Finally, log all the changes made during the execution of the PLAN in the appropriate documentation or change logs to maintain a record of the modifications for future reference. Check the checkboxes in the PLAN.md file for the tasks that have been completed. And also add the changes to the CHANGELOG.md file with a description of the changes made and the date of the change.
